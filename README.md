# บอร์ดอุปกรณ์สตูดิโอ (Studio Equipment Kanban Board)

เว็บจัดการอุปกรณ์สตูดิโอในรูปแบบ **บอร์ด Kanban** — ลากการ์ดอุปกรณ์ข้ามคอลัมน์
ซ้าย→ขวาเพื่อเปลี่ยนสถานะ โดยเก็บข้อมูลไว้ใน **Google Sheet**

## คอลัมน์สถานะ (ซ้าย → ขวา)

```
ใช้งานได้  →  เสียหายบางส่วน  →  ส่งซ่อม  →  เสียถาวร
```

## ฟีเจอร์

- 🗂️ **บอร์ด Kanban 4 คอลัมน์** — ลากการ์ดข้ามคอลัมน์เพื่อเปลี่ยนสถานะ
- 🎴 **การ์ดรวมชิ้น** — อุปกรณ์ชื่อเดียวกันในคอลัมน์เดียวกันจะรวมเป็นการ์ดเดียว
  แสดง **ชื่อ + จำนวนชิ้น**
- 🔽 **คลิกการ์ดเพื่อกาง** — เห็นอุปกรณ์แต่ละชิ้น และ **ลากทีละชิ้น** ไปคนละสถานะได้
- ⋮ **เมนูการ์ด** — ย้ายทั้งกลุ่ม / แก้ไขข้อมูล / ลบทั้งกลุ่ม (ใช้ได้บนมือถือที่ลากไม่ได้)
- ➕ เพิ่มอุปกรณ์ครั้งละหลายชิ้น (ระบุจำนวน)
- 🔎 ค้นหาชื่ออุปกรณ์
- ☁️ ข้อมูลอยู่บน Google Sheet — ทุกคนที่เปิดเว็บเห็นข้อมูลชุดเดียวกัน

> **โมเดลข้อมูล:** 1 แถวในชีต = อุปกรณ์ 1 ชิ้น (unit) — ทำให้ย้ายทีละชิ้นได้อิสระ
> แต่ละชิ้นเก็บ: **ชื่อ · หมวดหมู่ · สถานะ · ที่เก็บ · หมายเหตุ · วันที่อัปเดต**

## โครงสร้างไฟล์

| ไฟล์ | หน้าที่ |
|------|---------|
| `index.html` | หน้าเว็บหลัก (บอร์ด) |
| `styles.css` | สไตล์ |
| `app.js` | ตรรกะฝั่งเว็บ: บอร์ด, ลาก-วาง, เมนู, เรียก API |
| `config.js` | **ตั้งค่า** URL ของ API และรายการสถานะ (คอลัมน์) |
| `apps-script/Code.gs` | โค้ด Backend สำหรับวางใน Google Apps Script |

## วิธีติดตั้ง (ทำครั้งเดียว ~5 นาที)

### 1) สร้าง Google Sheet
ไปที่ [sheets.new](https://sheets.new) สร้างชีตใหม่ (ไม่ต้องใส่หัวตารางเอง ระบบสร้างให้อัตโนมัติ)

### 2) ติดตั้ง Backend (Apps Script)
1. ในชีต ไปที่ **Extensions → Apps Script**
2. ลบโค้ดเดิม แล้วคัดลอกเนื้อหาจาก [`apps-script/Code.gs`](apps-script/Code.gs) มาวาง → **Save**
3. **Deploy → New deployment** → ⚙️ เลือกชนิด **Web app**
   - **Execute as:** `Me`
   - **Who has access:** `Anyone`
   - **Deploy** แล้ว Authorize สิทธิ์ตามที่ระบบถาม
4. คัดลอก **Web app URL** (ลงท้ายด้วย `/exec`)

> ถ้าต้องการจำกัดเฉพาะคนในองค์กร ให้เลือก "Anyone within [องค์กร]" แทน (คนเปิดเว็บต้องล็อกอิน Google)

### 3) ตั้งค่าเว็บ
เปิด [`config.js`](config.js) แล้ววาง URL ลงในช่อง `API_URL`:

```js
window.APP_CONFIG = {
  API_URL: "https://script.google.com/macros/s/xxxxx/exec",  // ← วาง URL ที่นี่
  STUDIO_NAME: "บอร์ดอุปกรณ์สตูดิโอ",
  ...
};
```

### 4) เปิดใช้งาน
- ทดลองในเครื่อง: เปิดไฟล์ `index.html` ด้วยเบราว์เซอร์
- หรือ deploy ฟรีด้วย **GitHub Pages**: Push ขึ้น GitHub → **Settings → Pages** → เลือก branch → Save

## การใช้งานบนบอร์ด

- **เปลี่ยนสถานะทั้งกลุ่ม:** ลากการ์ดไปวางในคอลัมน์อื่น (หรือกด ⋮ → “ย้ายทั้งกลุ่มไป…”)
- **เปลี่ยนสถานะทีละชิ้น:** คลิกการ์ดให้กางออก → ลากชิ้นที่ต้องการไปคอลัมน์อื่น
  (หรือคลิกที่ชิ้น → เลือกสถานะปลายทาง)
- **เพิ่ม/แก้/ลบ:** ปุ่ม “+ เพิ่มอุปกรณ์” ด้านบน · ⋮ บนการ์ด · × บนแต่ละชิ้น

## ปรับแต่งเพิ่ม

- **เพิ่ม/แก้/สลับลำดับคอลัมน์สถานะ** — แก้ที่ `STATUSES` ใน `config.js` (ลำดับใน array = ลำดับคอลัมน์ซ้าย→ขวา)
- **เปลี่ยนชื่อบนหัวเว็บ** — แก้ `STUDIO_NAME` ใน `config.js`

## อัปเกรดในอนาคต (ถ้าต้องการ)

- รองรับลาก-วางบนมือถือ (touch drag) เต็มรูปแบบ
- ประวัติการย้ายสถานะ / วันที่ส่งซ่อม
- รูปอุปกรณ์ / QR code ต่อชิ้น
- ระบบล็อกอิน + สิทธิ์แก้ไข
